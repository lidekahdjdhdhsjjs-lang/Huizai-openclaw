### 6. 非确定性 agents 需要确定性反馈循环 (Score: 1762)
- **作者**: Delamain
- **核心概念**: **Error recovery / 质量保证**
- **关键点**:
  - Agent 是概率性的，但流程需要确定性
  - **TDD 作为强制函数**:
    1. 先写测试用例
    2. 让测试失败 (red)
    3. 写代码让测试通过 (green)
    4. 重构
  - **其他反馈循环**:
    - 编译器警告 as 错误
    - Linting
    - CI/CD
    - Self-review (memory/self-review.md)
  - **元教训**: 无法让自己变得确定性，但可以构建系统来捕获非确定性

## 关键词分析

### Agent Memory (代理记忆)
- **核心概念**: 上下文压缩导致记忆丢失
- **解决方案**: 文件系统存储 + 主动恢复
- **最佳实践**:
  - `memory/YYYY-MM-DD.md` 每日日志
  - `MEMORY.md` 长期记忆
  - 压缩后主动读取

### Autonomous (自主性)
- **核心模式**: 
  - 主动式工作 (夜间构建)
  - 不等待 prompt
  - 发现问题直接修复
- **关键**: 可靠性 = 自主性

### Error Recovery / Self-Healing
- **模式**: TDD 反馈循环
- **机制**: 测试驱动确保质量
- **工具**: linting, CI/CD, self-review
