# debug-pro - 调试诊断专家

## 诊断流程

### 1. 症状收集
```
- 错误信息
- 错误代码
- 发生时间
- 上下文环境
- 最近的操作
```

### 2. 根因分析
```
- 是确定性错误还是暂时性错误？
- 是首次发生还是重复发生？
- 是否有模式可循？
- 最近有什么变化？
```

### 3. 快速检查清单

#### 网络问题
- [ ] ping目标服务器
- [ ] curl测试端点
- [ ] 检查代理设置
- [ ] 检查SSL证书

#### 认证问题
- [ ] 检查token是否过期
- [ ] 验证权限配置
- [ ] 检查API密钥

#### 资源问题
- [ ] 检查磁盘空间
- [ ] 检查内存使用
- [ ] 检查进程状态
- [ ] 查看系统日志

#### 代码问题
- [ ] 最小复现步骤
- [ ] 查看最近改动
- [ ] 检查参数格式
- [ ] 验证输入数据

### 4. 常用诊断命令

```bash
# 网络诊断
curl -v <url>
traceroute <host>
nslookup <domain>

# 进程诊断
ps aux | grep <process>
top -b -n 1
lsof -i :<port>

# 日志诊断
tail -f /var/log/syslog
journalctl -xe
tail -n 100 <app-log>

# 资源诊断
df -h
free -m
cat /proc/cpuinfo

# 网络诊断
netstat -tulpn
ss -s
```

### 5. 输出格式

```markdown
## 诊断报告

**症状**: [描述问题]

**分析**: [可能的原因]

**检查结果**:
- [ ] 检查项1: 结果
- [ ] 检查项2: 结果

**结论**: [确定的根因]

**建议修复**: [步骤]
```
